# ADR-006: DB Topology — Shared PostgreSQL with Separate Schemas

## Context
Each microservice should have exclusive ownership of its data. The choice was between a separate PostgreSQL instance per service versus a shared instance with logical isolation.

## Decision
Single PostgreSQL instance, with a separate schema per service: `submission`, `decisions`, `approvals`, `payments`, `audit`. Each service connects only to its own schema. No service accesses another service's schema directly.

## Alternatives Considered

**Separate PostgreSQL instance per service (5 containers) — rejected for this project.**
This is the more correct architectural approach (full isolation, independent scaling), but adds significant memory and operational overhead that was not justified given the project's scope and timeline.

## Consequences

**What we gain:**
- Full logical ownership (separate migrations per service)
- Significant resource savings
- Simple `docker compose up` experience

**What we give up:**
- A failure in the PostgreSQL instance itself affects all services simultaneously
- In a real production system at scale, separate instances would be preferable

# ADR-006: טופולוגיית DB — Postgres משותף עם סכמות נפרדות

## הקשר
לכל מיקרו-שירות אמורה להיות בעלות בלעדית על הנתונים שלו. היה צריך להחליט בין מופע Postgres נפרד לכל שירות לבין מופע משותף עם הפרדה לוגית.

## ההחלטה
מופע PostgreSQL אחד, עם סכמה נפרדת לכל שירות: `submission`, `decisions`, `approvals`, `payments`, `audit`. כל שירות מתחבר רק לסכמה שלו. אף שירות לא ניגש ישירות לסכמה של שירות אחר.

## חלופות שנשקלו

**מופע Postgres נפרד לכל שירות (5 קונטיינרים) — נדחתה לצורך הפרויקט הזה.**
זו הגישה הנכונה יותר ארכיטקטונית (בידוד מלא, סקיילינג עצמאי), אבל מוסיפה תקורת זיכרון ותחזוקה משמעותית שלא הצדיקה את עצמה בהיקף ובזמן של הפרויקט.

## תוצאות

**מה מרוויחים:**
- בעלות לוגית מלאה (migrations נפרדים לכל שירות)
- חיסכון משמעותי במשאבים
- פשטות בהרצה עם `docker compose up`

**מה מוותרים עליו:**
- תקלה במופע ה-Postgres עצמו משפיעה על כל השירותים יחד
- בפרודקשן אמיתי בקנה מידה גדול, מופעים נפרדים היו עדיפים
