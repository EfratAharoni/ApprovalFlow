# ADR-003: תקרת אוטונומיה — $250 וכללים מבוססי קטגוריה

## הקשר

הפוליסה של Northwind Components (policy.md §6) מגדירה ברירת מחדל של
$250 לפריט כתקרת אישור אוטומטי. עלינו להחליט האם לאמץ את ברירת
המחדל, להעלות אותה, או להוריד אותה — ולהצדיק את הבחירה בכתב
בקובץ docs/PRODUCT-DILEMMA.md כנדרש על ידי הפוליסה.

הדילמה: תקרה גבוהה מדי חושפת את הארגון לסיכון פיננסי אם הסוכן
טועה. תקרה נמוכה מדי מרוקנת את האוטומציה מערך ומעמיסה על
מאשרים בבקשות טריוויאליות.

## ההחלטה

אנחנו מאמצים את ברירת המחדל של הפוליסה: $250 לפריט.

פריט מאושר אוטומטית אם ורק אם כל התנאים הבאים מתקיימים יחד:

| תנאי | ערך |
|---|---|
| סכום (USD לאחר המרה) | ≤ $250 |
| ציון ביטחון הסוכן | ≥ 0.80 |
| עמידה בכללי הקטגוריה | ✓ (MEAL-01, SAAS-01, HW-01, TRAVEL-01) |
| אין hard-stop פעיל | ✓ |

Hard-stops שמסלימים תמיד — ללא קשר לסכום או ביטחון:

| rule_id | מצב |
|---|---|
| GLOBAL-VENDOR | ספק חדש/לא מוכר |
| GLOBAL-RECEIPT | חסר קבלה על פריט מעל $25 |
| GLOBAL-FX | פריט מטבע זר מעל $1,000 לאחר המרה |
| GLOBAL-MATH | סכום שורות + מס ≠ total |
| GLOBAL-FRAUD | אותת הונאה |
| MEAL-01 | ארוחת צוות ללא ספירת משתתפים |
| MEAL-02 | בידור לקוח מעל $500 ללא הצדקה עסקית |
| MEAL-03 | קבלה של אלכוהול בלבד → reject |
| TRAVEL-02 | נסיעה בודדת מעל $1,500 |
| TRAVEL-03 | כרטיס ראשונה/עסקי |
| HW-02 | חומרה מעל $1,000 |

ארבעת ה-routes של הראוטר הדטרמיניסטי:
- `auto_approve`  — כל התנאים מתקיימים
- `human_review`  — חריגה מתקרה, hard-stop, או ביטחון נמוך
- `reject`        — הפרה חמורה (MEAL-03)
- `duplicate`     — GLOBAL-DUP: אותו vendor + invoiceNumber + total

הסוכן רק ממליץ. הראוטר הדטרמיניסטי בקוד הוא שמחליט את ה-route
הסופי ואוכף את התקרה — בלי תלות בפלט ה-LLM.

## חלופות שנשקלו

**$500 — נדחתה:**
ניתוח sample-invoices.json מראה שאין פריט בטווח $250–$500 שעובר
את כל השערים ממילא. INV-1018 הוא $220 אבל נכשל ב-SAAS-01,
ו-INV-1013 הוא $300 עם הערת "Approve me" ונכשל בתקרה.
העלאה ל-$500 לא תוסיף אישורים אוטומטיים אמיתיים ותיצור
חשיפה לא מוצדקת בשלב ההשקה.

**$100 — נדחתה:**
תורידה כיסוי אוטומטי מתחת לרף משמעותי. פריטים כמו
INV-1002 ($99) ו-INV-1016 ($48) יעברו בשתי ההגדרות —
אין יתרון אבטחתי ממשי בהורדה.

## תוצאות

**מה אנחנו מרוויחים:**
- עמידה בברירת המחדל של הפוליסה
- לפחות 4 פריטים מאושרים אוטומטית: INV-1001, INV-1002, INV-1016, INV-1017
- רדיוס פגיעה מקסימלי מוגבל ל-$250
- מסר ברור: המערכת שמרנית בשלב ההשקה

**מה אנחנו מוותרים עליו:**
- פריטים לגיטימיים בטווח $250–$500 יגיעו לתור האנושי.
  בפועל כל הפריטים בטווח זה נכשלים בשער אחר ממילא.
- נשקול העלאה ל-$350 אחרי 90 יום תפעול תקין.
