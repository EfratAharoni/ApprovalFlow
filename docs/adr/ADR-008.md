# ADR-008 — Hybrid BM25 + Vector Retrieval for Policy Lookup

## Context / הקשר

The AI agent uses `fetch_policy_clause(rule_id)` to look up policy text during its tool-calling loop.
This requires the LLM to already know the correct rule ID — fine for exact matches, but brittle when
the submission description uses vocabulary that differs from the policy text.

**Documented failure case (vector-only):**
A free-text query of `"team happy hour beverages"` failed to surface `MEAL-03`
(`"Alcohol-only receipts are not reimbursable"`) because there is zero lexical overlap between
the words *beverages / happy hour* and *alcohol-only / receipt*.
A purely semantic (vector) retrieval was also unreliable: the small embedding model used
(`all-MiniLM-L6-v2`) occasionally placed `MEAL-03` outside the top-3 for this query.

## Decision / החלטה

We replaced the direct-key lookup with a `PolicyIndex` class (`app/rag.py`) that supports
**hybrid retrieval**:

| Component | Handles | Weight |
|---|---|---|
| BM25 (rank_bm25) | Keyword / lexical overlap | 0.5 |
| Vector (sentence-transformers) | Semantic similarity | 0.5 |
| RRF-style fusion | Combined ranking | normalised sum |

Each score is min-max normalised to `[0, 1]` before fusion so neither signal dominates.

## Alternatives Considered / חלופות שנשקלו

| Alternative | Reason rejected |
|---|---|
| Vector-only | Missed `"team happy hour beverages" → MEAL-03` (semantic drift without lexical anchor) |
| BM25-only | Missed semantic queries with no shared vocabulary |
| Exact-ID lookup (current `fetch_policy_clause`) | Requires the LLM to know the ID in advance — breaks on novel phrasing |
| Fine-tuned domain embedding model | Overkill for 18 policy clauses; adds training complexity and infra |

## Consequences / השלכות

- `rank_bm25==0.2.2` and `sentence-transformers==3.0.1` added to `requirements.txt` (~200 MB added to Docker image; acceptable for the AI service layer)
- `PolicyIndex` is built once at module import (`policy_index` singleton) and held in memory — no DB needed for 18 clauses
- `fetch_policy_clause` tool is unchanged and still available for exact-ID lookups
- `tests/test_rag.py` added; covers keyword path, semantic-drift path, top-k contract, and custom-chunk injection

## עדכון: Hybrid BM25 + Vector (2026-07-01)

לאחר זיהוי הכשל הסמנטי, עברנו ל-hybrid retrieval.
BM25 מטפל ב-keyword overlap,
vector search מטפל בסמנטיקה,
fusion score משלב את שניהם במשקל שווה (0.5/0.5).
תוצאה: "team happy hour beverages" מוצא MEAL-03 ב-top-3 (היה נכשל לפני).
