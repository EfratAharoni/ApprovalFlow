# ADR-001: בחירת שפה — Python

## הקשר

היה צריך לבחור שפה לכל 6 השירותים. הבחירה משפיעה על מהירות הפיתוח,
התאמה ל-Dapr, ויכולת לבנות אגנט AI.

## ההחלטה

Python 3.11 + FastAPI לכל השירותים.

## חלופות שנשקלו

**TypeScript/Node.js — נדחתה:**
תמיכת Dapr טובה גם כן, אבל האקוסיסטם של AI Engineering (LiteLLM,
structured output, function calling) בנוי קודם כל סביב Python.

**C# — נדחתה:**
תמיכת Dapr מצוינת (Microsoft Agent Framework), אבל פחות נוח לבניית
אגנט מהירה בזמן הקצר שעמד לרשותי.

## תוצאות

**מה אנחנו מרוויחים:**
- ספריות AI בוגרות (LiteLLM, Pydantic לـ-structured output)
- קצב פיתוח מהיר
- type hints + docstrings הופכים אוטומטית לסכמות JSON עבור function calling

**מה אנחנו מוותרים עליו:**
- ביצועים נמוכים יותר מ-Go/C#, אבל לא קריטי בהיקף הפרויקט.

# ADR-001: Language Choice — Python

## Context

A language had to be chosen for all 6 services. The choice affects development speed,
Dapr compatibility, and the ability to build an AI agent.

## Decision

Python 3.11 + FastAPI for all services.

## Alternatives Considered

**TypeScript/Node.js — rejected:**
Dapr support is also good, but the AI Engineering ecosystem (LiteLLM, structured output,
function calling) is built primarily around Python.

**C# — rejected:**
Excellent Dapr support (Microsoft Agent Framework), but less convenient for rapidly
building an AI agent within the short timeframe available.

## Consequences

**What we gain:**
- Mature AI libraries (LiteLLM, Pydantic for structured output)
- Fast development pace
- type hints + docstrings automatically become JSON schemas for function calling

**What we give up:**
- Lower performance than Go/C#, but not critical at this project's scale.
