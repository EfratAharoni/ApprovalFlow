# ADR-001: Language Choice — Python 3.11 + FastAPI

## Context
A language had to be chosen for all 6 services. The choice affects development velocity, Dapr compatibility, and the ability to build an AI agent with function calling and structured output.

## Decision
Python 3.11 + FastAPI for all services.

## Alternatives Considered

**TypeScript/Node.js — rejected.**
Dapr support is equally good, but the AI Engineering ecosystem (LiteLLM, Pydantic for structured output, function calling) is built first around Python. Type hints + Pydantic models become JSON schemas for function calling automatically — no extra code required.

**C# — rejected.**
Excellent Dapr support (Microsoft Agent Framework), but less convenient for rapid agent development in the time available, and AI libraries are less mature.

## Consequences

**What we gain:**
- Mature AI libraries (LiteLLM, Pydantic)
- Fast development velocity
- Type hints become JSON schemas automatically
- MockAgent is injectable for offline testing

**What we give up:**
- Lower raw throughput than Go/C# under high load
- Not critical at this project's scale

# ADR-001: בחירת שפה — Python 3.11 + FastAPI

## הקשר
היה צריך לבחור שפה לכל 6 השירותים. הבחירה משפיעה על מהירות הפיתוח, התאמה ל-Dapr, ויכולת לבנות אגנט AI עם function calling ו-structured output.

## ההחלטה
Python 3.11 + FastAPI לכל השירותים.

## חלופות שנשקלו

**TypeScript/Node.js — נדחתה.**
תמיכת Dapr טובה גם כן, אבל האקוסיסטם של AI Engineering (LiteLLM, Pydantic לstructured output, function calling) בנוי קודם כל סביב Python. type hints + docstrings הופכים אוטומטית לסכמות JSON עבור function calling — ללא קוד נוסף.

**C# — נדחתה.**
תמיכת Dapr מצוינת (Microsoft Agent Framework), אבל פחות נוח לבניית אגנט מהירה בזמן הקצר שעמד לרשותי, וספריות AI פחות בשלות.

## תוצאות

**מה מרוויחים:**
- ספריות AI בוגרות (LiteLLM, Pydantic)
- קצב פיתוח מהיר
- type hints הופכים ל-JSON schemas אוטומטית
- MockAgent ניתן להזרקה לבדיקות offline

**מה מוותרים עליו:**
- ביצועים נמוכים יותר מ-Go/C# תחת עומס גבוה
- לא קריטי בהיקף הפרויקט
