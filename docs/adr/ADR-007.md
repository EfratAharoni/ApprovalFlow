# ADR-007: API Gateway — Nginx מול Proxy מותאם אישית

## הקשר

דרישת M6 — נקודת כניסה חיצונית יחידה עם rate-limiting. צריך להחליט
על הטכנולוגיה.

## ההחלטה

Nginx עם מודול `limit_req` מובנה.

## חלופות שנשקלו

**Gateway מותאם אישית בـ-Python/FastAPI — נדחתה:**
כתיבת gateway עצמאי דורשת מימוש ידני של rate-limiting, routing, וטיפול
בכשלים — כל אלה כבר פתורים ובדוקים-בקרב בـ-Nginx. למרות שـ-FastAPI
gateway היה נותן יותר גמישות (למשל לוגיקת ניתוב מורכבת יותר), הצרכים
שלנו (ניתוב סטטי + rate-limiting) פשוטים מספיק שـ-Nginx הוא הפתרון
המתאים והבטוח יותר.

**שירותי Gateway מנוהלים (Azure API Management, AWS API Gateway) — נדחו:**
דורשים תלות בענן ספציפי וחורגים מהדרישה "ללא שירותים בתשלום" של הפרויקט.

## תוצאות

**מה אנחנו מרוויחים:**
- יציבות מוכחת
- ביצועים גבוהים
- קונפיגורציה פשוטה דרך קובץ אחד
- אין תלות בענן

**מה אנחנו מוותרים עליו:**
- גמישות תכנותית — אם נצטרך בעתיד לוגיקת ניתוב דינמית (למשל לפי
  תוכן הـ-payload, A/B testing), נצטרך לעבור לـ-gateway מתוכנת
  או להוסיף שכבה נוספת.

# ADR-007: API Gateway — Nginx vs. Custom Proxy

## Context

Requirement M6 — a single external entry point with rate-limiting. We needed to
decide on the technology.

## Decision

Nginx with the built-in `limit_req` module.

## Alternatives Considered

**Custom gateway in Python/FastAPI — rejected:**
Writing a standalone gateway requires manual implementation of rate-limiting, routing,
and error handling — all of which are already solved and battle-tested in Nginx.
While a FastAPI gateway would offer more flexibility (e.g., more complex routing logic),
our needs (static routing + rate-limiting) are simple enough that Nginx is the more
appropriate and safer choice.

**Managed gateway services (Azure API Management, AWS API Gateway) — rejected:**
Require cloud-vendor lock-in and exceed the project's "no paid services" requirement.

## Consequences

**What we gain:**
- Proven stability
- High performance
- Simple configuration via a single file
- No cloud dependency

**What we give up:**
- Programmatic flexibility — if we ever need dynamic routing logic (e.g., based on
  payload content, A/B testing), we would need to switch to a programmatic gateway
  or add an additional layer.
